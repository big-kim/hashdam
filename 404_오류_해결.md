# 404 오류 해결 완료

## 발생한 문제

배포된 사이트에서 다음과 같은 404 오류가 발생했습니다:
1. `images/group-4273188750.svg` - 404 오류
2. `site.webmanifest` - 404 오류 및 manifest fetch 실패

## 해결한 내용

### 1. site.webmanifest 파일 수정
`public/site.webmanifest` 파일의 아이콘 경로에 basePath를 추가했습니다:

**수정 전:**
```json
"icons": [
  {
    "src": "images/logo-hash.png",
    "sizes": "192x192",
    "type": "image/png"
  }
]
```

**수정 후:**
```json
"icons": [
  {
    "src": "/hashdam/images/logo-hash.png",
    "sizes": "192x192",
    "type": "image/png"
  }
]
```

### 2. start_url 추가
manifest 파일에 올바른 start_url을 추가했습니다:
```json
"start_url": "/hashdam/",
```

## 빌드 확인

빌드는 성공적으로 완료되었으며, 생성된 HTML에서 이미지 경로와 manifest 링크가 올바르게 설정되어 있습니다.

## 다음 단계

변경사항을 GitHub에 배포해야 합니다:

1. 변경사항 커밋:
```bash
git add public/site.webmanifest
git commit -m "Fix manifest paths for GitHub Pages deployment"
git push
```

2. GitHub Actions가 자동으로 빌드하고 배포합니다.

3. 배포 완료 후 브라우저에서 확인:
   - https://big-kim.github.io/hashdam
   - 개발자 도구 콘솔에서 404 오류가 사라졌는지 확인

## 참고사항

- `group-4273188750.svg` 파일은 `public/images/` 폴더에 올바르게 위치해 있습니다
- Next.js의 `getImagePath` 함수가 프로덕션 환경에서 `/hashdam` basePath를 올바르게 추가합니다
- manifest 파일의 모든 경로는 GitHub Pages 배포 환경에 맞게 절대 경로로 설정되었습니다
